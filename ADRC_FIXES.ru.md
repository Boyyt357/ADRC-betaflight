# Правки надёжности ADRC — призыв к тестерам

[English](ADRC_FIXES.md) | **Русский**

Этот форк добавляет серию **небольших независимых** правок надёжности поверх
[`Boyyt357/ADRC-betaflight`](https://github.com/Boyyt357/ADRC-betaflight). Они появились по
итогам построчной сверки реализации ADRC в `src/main/flight/pid.c` с теорией ADRC/LADRC.
Математическое ядро (LESO + виртуальный PD + компенсация возмущения, bandwidth-параметризация
Gao 2003) реализовано точно; эти правки бьют по **стыку надёжности**, где ADRC встречается с
легаси-обвязкой Betaflight.

> ⚠️ **Все правки НЕ протестированы на реальном железе.** У меня нет аппарата, чтобы их
> облетать. Каждая правка — **отдельный коммит**, поэтому можно собрать с любой из них или без
> (`git revert <sha>`, чтобы убрать одну). **Пожалуйста, облетайте и отписывайтесь в issues.**

## Что сообщать
- Аппарат (размер, вес, моторы/пропы, целевая плата) и ваши значения ADRC P/I/D (wc/wo/b0).
- С какими коммитами собирали (или «все»).
- Поведение: арм/раскрутка, висение, резкие манёвры, prop wash, ветер, восстановление после
  сброса газа / десинка, любые осцилляции или нагрев моторов.
- По возможности — лог blackbox.

## Коммиты

| Правка | Тема коммита | Что меняет | Риск |
| :-- | :-- | :-- | :-- |
| #3 дефолты | `re-derive default P/I/D as ADRC bandwidths` | Дефолтные P/I/D были неконвертированными стоковыми PID-числами (wo/wc≈1.78); теперь wc=10, wo=110, b0=100 (yaw wo=80) по проверенному 5″-тюну, с явным yaw b0. | Низкий (только дефолты) |
| #1 anti-windup | `add anti-windup clamp on the ESO disturbance estimate z3` | Ограничивает z3, чтобы член компенсации `z3/b0` оставался в пределах per-axis pidsum-лимита; убирает запаздывание восстановления после насыщения моторов. | Средний |
| #2 подача в наблюдатель | `feed the ESO the saturation-limited command` | Член `b0*u` в ESO теперь использует pidsum, зажатый по per-axis лимиту (то, что реально может дойти до объекта), а не сырой досатурационный pidsum. | Средний |
| #4 нулевой газ | `keep the ESO running at zero throttle` | На нулевом газе (без airmode) наблюдатель сбрасывался каждый цикл; теперь он продолжает интегрировать, обнуляется только легаси-аккумулятор I, и оценка возмущения переживает раскрутку. Настоящие disarm/overflow по-прежнему полностью сбрасывают. | Средний |
| #5 только ядро | `EXPERIMENTAL — feed ESO only the ADRC core (exclude F/S)` | Подаёт в наблюдатель только P+I+D, исключая feedforward F и S-term (они не делятся на b0). **Спорно** — F/S тоже прикладываются к объекту. Откатите этот один коммит, чтобы вернуть поведение #2; летайте оба варианта подряд для сравнения. | Высокий / неоднозначный |

## Как собрать с правкой или без неё
```
git log --oneline          # найдите SHA коммитов
git revert <sha>           # убрать одну правку (создаёт revert-коммит)
# затем соберите как обычно: make <TARGET>   (см. README, раздел «Сборка ADRC-Betaflight»)
```

## Контекст
Полная сверка теории и реализации (что совпадает с каноническим LADRC, что является сознательным
упрощением, что отклоняется, и какие риски надёжности закрывают эти коммиты) лежит вне дерева
прошивки, в заметках мейнтейнера; краткая версия каждого замечания — в сообщениях коммитов.
Вопросы и результаты → **открывайте issue**.
